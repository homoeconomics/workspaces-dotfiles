# Project Guidelines

## Git Branch Naming

When creating a new git branch, use the format:

```
daniel.zhou/<JIRA-ticket-ID>-<branch-name>
```

Example: `daniel.zhou/CWS-6211-signiture-update`

## Pull Request Descriptions

When creating a PR, include a diagram of the service architecture in the PR description if it helps reviewers understand the changes. Use Mermaid or any other diagram tool supported by GitHub.
